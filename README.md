# Шашки — Prolog + MinMax Alpha-Beta

## Структура проекту

```
prolog/
  checkers.pl         — логіка гри (дошка, генерація ходів, оцінка)
  alphabeta.pl        — Компʼютер: алгоритм MinMax з Alpha-Beta відсіканням
  server.pl           — HTTP-сервер (порт 5500)
  index.html          — веб-інтерфейс
python/
  checkers_python.py  — порівняльна реалізація (Python, консоль)
  checkers_gui.py     — порівняльна реалізація (Python, GUI)
haskell/
  Checkers.hs         — порівняльна реалізація (Haskell, бібліотека)
  checkers_haskell.hs — порівняльна реалізація (Haskell, консоль)
  test_checkers_haskell.hs — тести для Haskell-реалізації
```

## Запуск Prolog-версії (веб-інтерфейс)

### Вимоги
- SWI-Prolog ≥ 8.0: https://www.swi-prolog.org/download/stable

### Кроки

```bash
# 1. Запустити сервер
cd prolog
swipl server.pl

# Виведе: Starting checkers server on http://localhost:5500/

# 2. Відкрити у браузері
http://localhost:5500/
```

> Зупинити сервер: Ctrl+C у терміналі

## Запуск Python-версії (консоль)

```bash
cd python
python checkers_python.py
```

Формат ходу: `рядок_від кол_від рядок_до кол_до` (1-based)
Наприклад: `3 2 4 3`

## Запуск Haskell-версії (консоль)

### Вимоги
- GHC ≥ 9.0: https://www.haskell.org/ghcup/

### Кроки

```bash
cd haskell

# 1. Скомпілювати
ghc -O2 Checkers.hs checkers_haskell.hs -o checkers_haskell

# 2. Запустити
./checkers_haskell
```

Формат ходу: `рядок_від кол_від рядок_до кол_до` (1-based)
Наприклад: `3 2 4 3`

## Правила

- Чорні ходять першими (рух вниз, рядки 1→8)
- Обов'язкове взяття; ланцюгові взяття
- Дамка з'являється при досягненні крайнього рядка
- Нічия після 100 напівходів

## Алгоритм Компʼютера

- MinMax з Alpha-Beta відсіканням, глибина: 5 напівходів (дебют, перші 6 ходів) / 8 напівходів (решта гри)
- Оцінка: шашка = 100 очок, дамка = 300 очок; бонус за центр (+10) та просування (+5/рядок)
- Відсікання: α-відсікання (MIN-вузол), β-відсікання (MAX-вузол)

## Чому не CLP?

Логіка гри — процедурний пошук (генерація ходів + MinMax з Alpha-Beta),
а не задоволення обмежень. Позиції дошки повністю визначені на кожному кроці,
і ходи генеруються явним перебором позицій фігур і напрямків. Поширення
обмежень не дало б суттєвого скорочення, оскільки простір пошуку визначається
деревом гри, а не доменами змінних.
